# P67 Local Development Install

Set up the P67 local development environment from scratch. Follows the steps in `QUICKSTART.md` with built-in diagnostics and troubleshooting.

## Prerequisites

Before starting, run the prerequisite checker:

```bash
bash p67/.claude/skills/install/tools/check-prereqs.sh
```

Required tools:
- **Node.js** v20+ (`node --version`)
- **Bun** v1.1+ (`bun --version`)
- **pnpm** v10+ (`pnpm --version`)
- **Docker** and **Docker Compose** (`docker --version`, `docker compose version`)
- **openssl** (for encryption key generation)

If any are missing, install them before proceeding:
- Node.js: https://nodejs.org/ or `brew install node`
- Bun: `curl -fsSL https://bun.sh/install | bash`
- pnpm: `npm install -g pnpm`
- Docker: https://docs.docker.com/desktop/

## Step 1: Configure the Snowflake Connection

This generates the root `Makefile` from `Makefile.template` by substituting your Snow CLI connection name.

```bash
cd p67 && ./configure.sh
```

This is interactive -- it lists your `snow` CLI connections and asks you to pick one. The result is written to `p67/Makefile`.

**Troubleshooting:**
- If `snow` CLI is not found: install Snowflake CLI (`pip install snowflake-cli-labs` or `brew install snowflake-cli`)
- If no connections listed: run `snow connection add` first
- If `jq` is not found: `brew install jq`

## Step 2: Start Local Services (`make dev`)

**Important:** You must run `./configure.sh` (Step 1) before this step. `make dev` requires the root `Makefile`, which is generated by `configure.sh`. Without it you'll get `make: No targets specified and no makefile found`.

```bash
cd p67 && make dev
```

This does the following in order:
1. Copies `.env.example` to `.env` in `packages/db/` and `services/controld/`
2. Generates an `ENCRYPTION_KEY` in the root `.env` if not already set
3. Runs `docker compose up --build -d` (builds controld image, starts PostgreSQL + controld containers)
4. Waits for PostgreSQL to be healthy
5. Runs Prisma migrations (`packages/db` deploy + generate)
6. Tails logs from all containers

The services run in Docker in the background. You can `Ctrl+C` out of the log tail without stopping them. To view logs again later: `docker compose logs -f`.

**Troubleshooting:**

### `make: No targets specified and no makefile found`
You skipped Step 1. Run `./configure.sh` first to generate the root Makefile.

### Docker not running
```
Cannot connect to the Docker daemon
```
Start Docker Desktop or the Docker daemon.

### Port 5432 already in use
```
Bind for 0.0.0.0:5432 failed: port is already allocated
```
Another PostgreSQL instance is using port 5432. Stop it:
```bash
# Find what's using the port
lsof -i :5432
# If it's a local postgres
brew services stop postgresql
# Or kill the process
kill <PID>
```

### Port 3002 already in use
Same approach as above but for port 3002:
```bash
lsof -i :3002
kill <PID>
```

### Prisma migration fails
```
Error: P1001: Can't reach database server
```
PostgreSQL may not be ready yet. Check:
```bash
docker compose ps   # Is postgres running?
docker compose logs postgres  # Any errors?
```

### Docker build fails (node_modules / pnpm)
```
ERR_PNPM_LOCKFILE_MISSING
```
Run `pnpm install` at the repo root first, then retry `make dev`.

### ENCRYPTION_KEY issues
The `make dev` target auto-generates this. If you need to regenerate:
```bash
openssl rand -base64 32
```
Then set it in `p67/.env` as `ENCRYPTION_KEY=<value>`.

## Step 3: Build and Install the CLI

In a **separate terminal**:

```bash
cd p67 && make install-cli
```

This does:
1. `pnpm install --filter "@p67/workflow-sdk" --filter "@p67/cli"` -- installs deps for the SDK and CLI packages
2. `pnpm --filter "@p67/workflow-sdk" run build` -- builds the workflow SDK (CLI dependency)
3. `make clean install -C ./tools/p67-cli` -- builds the CLI binary with Bun, then runs `install.sh`

The `install.sh` script will:
- Prompt for an install directory (default: `/usr/local/bin`)
- Create a symlink from `p67` to `tools/p67-cli/bin/p67`
- Warn if the directory is not in `$PATH`

**Troubleshooting:**

### `bun` not found during CLI build
The CLI uses Bun as its runtime. Install Bun:
```bash
curl -fsSL https://bun.sh/install | bash
```

### Permission denied on `/usr/local/bin`
Choose a different directory like `~/.local/bin` when prompted, and ensure it's in your `$PATH`:
```bash
export PATH="$PATH:$HOME/.local/bin"
```

### Build errors (TypeScript)
Try a clean build:
```bash
pnpm install
pnpm -r build
```

## Step 4: Configure a CLI Connection

```bash
p67 connection add p67 --endpoint http://localhost:3002
```

You'll be prompted for a Snowflake PAT. **For local development, enter any value** (e.g., `dummy`).

**Troubleshooting:**

### Connection refused
controld is not running. Check:
```bash
docker compose ps
docker compose logs controld
```

### Verify the connection works
```bash
p67 workflow list
```
Should return an empty list (not an error).

## Step 5: First Workflow (Verification)

```bash
mkdir -p /tmp/myworkflow && cd /tmp/myworkflow
p67 init
p67 build
p67 workflow deploy
p67 workflow run
```

If all four commands succeed, the install is complete.

**Troubleshooting:**

### `p67 init` fails
- Ensure `p67` is in your `$PATH` (`which p67`)
- Ensure controld is running and the connection is configured (`p67 connection list`)

### `p67 build` fails
- Check Node.js version: `node --version` (need v20+)
- Check that `pnpm` is available

### `p67 workflow deploy` fails with connection error
- Verify controld is running: `curl http://localhost:3002/api/health` (or similar)
- Re-check connection: `p67 connection list`

### `p67 workflow run` fails
- Check controld logs: `docker compose -f p67/compose.yaml logs controld`

## Diagnostic Tool

If something is wrong and you're not sure what, run the doctor script:

```bash
bash p67/.claude/skills/install/tools/doctor.sh
```

This checks all prerequisites, Docker state, port availability, database connectivity, CLI installation, and connection configuration in one pass.

## Full Reset

To tear down everything and start fresh:

```bash
bash p67/.claude/skills/install/tools/reset.sh
```

This stops Docker containers, removes volumes, deletes generated `.env` files and the root `Makefile`, and cleans build artifacts. You can then re-run the install from Step 1.
