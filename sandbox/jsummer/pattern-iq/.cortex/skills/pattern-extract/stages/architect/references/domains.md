# Logical Domains

| Domain | Description | Signals |
|---|---|---|
| `data-ingestion` | Loading, staging, importing raw data | COPY INTO, stages, pipes, loaders, ingest |
| `data-transformation` | Cleaning, transforming, modeling data | dbt models, SQL transforms, ETL, ELT |
| `llm-orchestration` | LLM agents, chains, prompt management | LangChain, LangGraph, ChatModel, agent, prompt |
| `document-processing` | PDF parsing, OCR, text extraction, chunking | pdfplumber, tesseract, extract_text, chunk |
| `api-service` | REST/gRPC endpoints, request handling, middleware | FastAPI, Flask, endpoint, route, handler |
| `frontend-ui` | Streamlit apps, React components, UI state | st., useState, pages, components, layout |
| `auth-identity` | Authentication, authorization, token management | OAuth, JWT, RBAC, login, session, token |
| `search-retrieval` | Search services, RAG, vector stores, embeddings | Cortex Search, embeddings, retriever, index |
| `observability` | Logging, metrics, monitoring, tracing | logger, metrics, tracing, recorder, sink |
| `testing-quality` | Tests, fixtures, data quality checks, validations | pytest, assert, DMF, validate, fixture |
| `deployment-infra` | CI/CD, Docker, SPCS, GitHub Actions, deployment | Dockerfile, workflow, deploy, Makefile |
| `configuration` | Config files, environment setup, project metadata | settings, env, config, pyproject, manifest |
